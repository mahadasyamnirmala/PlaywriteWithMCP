🚨 Built an AI-powered test automation framework that eliminates manual bug reporting
As a QA Engineer transitioning towards SDET, I focused on solving a real problem:
👉 Reducing manual effort in testing + defect tracking
So I built an end-to-end automation pipeline using Playwright, JavaScript, and AI (MCP).

⚙️ Tech Stack:
Playwright | JavaScript | JIRA | AI (Model Context Protocol) | HTML Reporting

🔄 End-to-End Workflow:
📝 Test Planning & Design
	• Defined scope, test scenarios, and edge cases
	• Applied ECP & Boundary Value Analysis concepts
🧪 Automation Framework (Playwright)
	• Page Object Model (POM) design pattern
	• Data-driven testing approach
	• Reusable utilities for scalability
🚀 Test Execution
	• Cross-browser automation
	• Assertions & validations
	• Negative testing (SQL injection, invalid inputs, multilingual data)
📊 Reporting & Debugging
	• Interactive HTML reports
	• Execution logs, traces, screenshots
🤖 AI-Based Bug Creation (JIRA Integration)
	• Automatic defect creation on failure
	• Includes repro steps, logs, attachments
	• Improves defect quality & developer productivity

📊 Results / Impact:
	• ⏱️ ~80% reduction in manual bug reporting effort
	• ⚡ Faster regression cycles
	• 📈 Improved defect tracking accuracy
